# Merge AGENTS.md + CLAUDE.md and Extract Skills as FSX

**Status:** Draft

---

## Description

Merge `AGENTS.md` and `CLAUDE.md` into a single source of truth. Extract session-infrequent content into loadable skills. All generated `.md` files are produced by FsAgent `.fsx` scripts placed alongside their output.

### Goals

1. One canonical agent instructions file (generated as both `AGENTS.md` and `CLAUDE.md` from the same source).
2. Keep always-needed content in AGENTS.md; move occasional content to skills.
3. Improve the existing create-agent/create-command/create-skill skills with missing operations and better examples.
4. Add a new publish skill for release workflow.
5. All `.md` outputs are generated by `.fsx` files using the FsAgent DSL.

### File Layout

| Generator | Output |
|---|---|
| `./AGENTS.fsx` | `./AGENTS.md` + `./CLAUDE.md` |
| `./.agents/skills/create-agent.fsx` | `./.agents/skills/create-agent/SKILL.md` |
| `./.agents/skills/create-command.fsx` | `./.agents/skills/create-command/SKILL.md` |
| `./.agents/skills/create-skill.fsx` | `./.agents/skills/create-skill/SKILL.md` |
| `./.agents/skills/publish.fsx` | `./.agents/skills/publish/SKILL.md` |

### References

- Current `AGENTS.md` (80 lines) — general principles, tech stack, build/test, testing, repo overview, templating, coding guidelines, git conventions.
- Current `CLAUDE.md` (209 lines) — project overview, build commands, detailed architecture (core types, data flow, output formats, tool configuration table, writer options), test organization.
- `ARCHITECTURE.md` and `PROJECT.md` — already contain the detailed architecture and project info that CLAUDE.md duplicates.
- Existing skills in `.agents/skills/` — create-agent, create-command, create-skill (generated by `examples/opencode-skill-factory/opencode-skill-factory.fsx`).

### 2.1 — Improved `create-agent` skill

**Sections (4):**

1. **Boilerplate**

   ```fsharp
   open FsAgent.Agents
   open FsAgent.Prompts
   open FsAgent.Tools
   open FsAgent.Writers
   ```

2. **Pattern** — complete runnable example showing the common case
   ```fsharp
   let myAgent = agent {
       name        "my-agent"
       description "A helpful assistant."
       model       "claude-opus-4-5"
       tools       [Read; Write; Bash]
       prompt (prompt {
           role         "You are a helpful assistant."
           instructions "Answer concisely."
       })
   }
   
   let path = FileWriter.writeAgent myAgent AgentWriter.Opencode (Project ".") (fun _ -> ())
   printfn "Wrote %s" path
   ```

3. **Operations** — compact flat list (not a table)
   ```
   name : string (required)
   description : string
   model : string
   temperature : float
   maxTokens : float
   tools : Tool list
   disallowedTools : Tool list
   author : string
   version : string
   license : string
   prompt : Prompt (embed a reusable prompt)
   section : string * string
   import : string (code-block wrapped)
   importRaw : string (raw embed)
   meta : MetaBuilder (use kv, kvList, kvObj, kvListObj for custom frontmatter)
   
   Available tools: Write, Edit, Bash, Shell, Read, Glob, List, LSP, Skill, TodoWrite, TodoRead, WebFetch, WebSearch, Question, Todo, Custom "name"
   ```

## Phase 3 — Write AGENTS.fsx

Create the `AGENTS.fsx` script at the repo root that generates both `AGENTS.md` and `CLAUDE.md`.

### Approach

- Use `prompt { }` builder with `section` operations for each heading
- Render via `AgentWriter.renderPrompt` (produces markdown without YAML frontmatter)
- Write identical content to both `./AGENTS.md` and `./CLAUDE.md` via `System.IO.File.WriteAllText`
- Add a generated-file notice at the top pointing to `AGENTS.fsx`

### Script structure

```fsharp
#r "src/FsAgent/bin/Debug/netstandard2.0/FsAgent.dll"

open FsAgent.Prompts
open FsAgent.Writers

let agentsMd = prompt {
    section "FsAgent" "..."
    section "Key files" "..."
    section "General" "..."
    section "Tech Stack" "..."
    section "Build & Test" "..."
    // ... etc
}

let content = AgentWriter.renderPrompt agentsMd (fun _ -> ())

let header = "<!-- Generated by AGENTS.fsx — do not edit directly -->\n\n"
System.IO.File.WriteAllText("AGENTS.md", header + content)
System.IO.File.WriteAllText("CLAUDE.md", header + content)
printfn "Wrote AGENTS.md and CLAUDE.md"
```

## Phase 4 — Write Skill FSX Files

Create the 4 skill `.fsx` generators in `.agents/skills/`.

### Approach

- Each `.fsx` uses `skill { }` builder and `FileWriter.writeSkill`
- Target harness: `AgentWriter.Opencode` (writes to `.agents/skills/` by default with `AgentsFolder` variant)
- Scope: `Project` with path resolved relative to `__SOURCE_DIRECTORY__`
- Each script is self-contained and runnable with `dotnet fsi`

### Script structure (template for each)

```fsharp
#r "../../src/FsAgent/bin/Debug/netstandard2.0/FsAgent.dll"
#r "nuget: Fue, 2.2.0"  // Only if using template nodes

open FsAgent.Skills
open FsAgent.Writers

let projectRoot =
    System.IO.Path.GetFullPath(
        System.IO.Path.Combine(__SOURCE_DIRECTORY__, "..", ".."))

let mySkill = skill {
    name "skill-name"
    description "..."
    section "Boilerplate" "..."
    // ... content from Phase 2
}

let path =
    FileWriter.writeSkill mySkill AgentWriter.Opencode (Project projectRoot) (fun _ -> ())
printfn "Wrote %s" path
```
