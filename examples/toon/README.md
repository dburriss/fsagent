# toon

Demonstrates a complete agent definition with imports, metadata, custom tools, and custom footers.

This example shows how to build a richer agent that imports external data files, uses custom tools (not built-in tools), includes project metadata, and applies custom footer generation logic. It illustrates the full range of agent configuration options.

## Files

- `toon.fsx` - Script that generates the agent definition
- `toon-agent.md` - Output agent file with import data and mission strategy
- `toon-data.toon` - Sample data file imported by the agent
